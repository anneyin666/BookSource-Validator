# 生产环境部署指南

本文档介绍如何将阅读书源去重校验工具部署到云服务器，通过外网访问。

---

## 部署架构

```
┌─────────────────────────────────────────────────────────┐
│                      云服务器                            │
│                                                         │
│   ┌─────────────────────────────────────────────────┐   │
│   │              FastAPI 后端                        │   │
│   │         (http://0.0.0.0:8000)                   │   │
│   │                                                  │   │
│   │    ┌─────────────┐    ┌─────────────────────┐   │   │
│   │    │  API 路由   │    │  静态文件服务       │   │   │
│   │    │  /api/*     │    │  前端 Vue SPA       │   │   │
│   │    └─────────────┘    └─────────────────────┘   │   │
│   └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
                   外网访问 (http://服务器IP:8000)
```

**说明**：生产环境下，FastAPI 同时提供 API 服务和前端静态文件服务，无需 Nginx 反向代理即可运行。

## 部署步骤

### 1. 上传代码到服务器

```bash
# 方式一：Git 克隆（推荐）
git clone https://github.com/anneyin666/BookSource-Validator.git
cd BookSource-Validator

# 方式二：直接上传压缩包
# 本地打包后上传到服务器解压
```

### 2. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

### 3. 构建前端

```bash
cd ../frontend
npm install
npm run build
```

构建完成后会生成 `frontend/dist` 目录，包含所有静态文件。

### 4. 启动服务

**方式一：使用启动脚本（推荐）**

```bash
# 回到项目根目录
cd ..

# Windows
start-server.bat

# Linux/Mac
chmod +x start-server.sh
./start-server.sh
```

**方式二：手动启动**

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**方式三：后台运行（Linux）**

```bash
cd backend
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &
```

### 5. 验证部署

打开浏览器访问：

- **前端页面**：`http://服务器IP:8000`
- **API 文档**：`http://服务器IP:8000/docs`

## 配置说明

### 端口配置

默认端口为 8000，如需修改：

```bash
# 启动时指定端口
uvicorn app.main:app --host 0.0.0.0 --port 9000
```

### 防火墙配置

确保服务器防火墙开放对应端口：

```bash
# CentOS/RHEL
sudo firewall-cmd --zone=public --add-port=8000/tcp --permanent
sudo firewall-cmd --reload

# Ubuntu/Debian
sudo ufw allow 8000/tcp
sudo ufw reload

# 云服务器安全组
# 阿里云/腾讯云/AWS 等需要在控制台开放安全组端口
```

### CORS 配置

后端 CORS 已配置为允许所有来源（`["*"]`），适用于生产环境。如需限制：

修改 `backend/app/config.py`：

```python
CORS_ORIGINS: List[str] = ["http://yourdomain.com"]
```

## 生产优化

### 使用 Gunicorn（推荐）

Gunicorn 是生产级 WSGI 服务器，性能更好：

```bash
# 安装
pip install gunicorn

# 启动（4个工作进程）
cd backend
gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

### 使用 Systemd 服务（Linux）

创建系统服务，开机自启：

```bash
sudo nano /etc/systemd/system/booksource-validator.service
```

写入内容：

```ini
[Unit]
Description=BookSource Validator Service
After=network.target

[Service]
User=your_username
Group=your_group
WorkingDirectory=/path/to/BookSource-Validator/backend
ExecStart=/usr/bin/gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start booksource-validator
sudo systemctl enable booksource-validator  # 开机自启
```

### Nginx 反向代理（可选）

如需使用域名访问或 HTTPS，可配置 Nginx：

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 常见问题

### 1. 访问显示 502 Bad Gateway

**原因**：后端服务未启动

**解决**：检查服务是否运行

```bash
# 查看进程
ps aux | grep uvicorn

# 查看日志
tail -f app.log
```

### 2. 访问显示空白页

**原因**：前端未构建或构建路径错误

**解决**：

```bash
# 确认 dist 目录存在
ls frontend/dist/

# 重新构建
cd frontend && npm run build
```

### 3. API 请求跨域错误

**原因**：CORS 配置问题

**解决**：确认 `backend/app/config.py` 中 CORS_ORIGINS 包含 `"*"`

### 4. 外网无法访问

**原因**：防火墙未开放端口

**解决**：

1. 检查服务器防火墙
2. 检查云服务商安全组配置
3. 确认启动时使用 `--host 0.0.0.0`
