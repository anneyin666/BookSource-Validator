# 前端组件设计

## 1. 页面结构

当前前端由一个主页面 `App.vue` 统筹，按模式切换不同功能区。

```text
App.vue
├── ThemeToggle
├── MessageAlert
├── 单个处理模式
│   ├── FileUpload
│   ├── UrlInput
│   ├── ActionButtons
│   ├── ConcurrencySelector
│   ├── TimeoutSelector
│   └── FilterOptions
├── 批量处理模式
│   ├── BatchProcessor
│   ├── ConcurrencySelector
│   ├── TimeoutSelector
│   └── FilterOptions
├── 搜索校验模式
│   ├── FileUpload
│   └── SearchValidator
├── ValidationProgress
├── StatsCardGroup
├── FailedSources
├── 失败详情列表
└── DownloadButton
```

## 2. 核心状态

状态集中在 [frontend/src/composables/useSources.js](../frontend/src/composables/useSources.js)。

```javascript
state = {
  file,
  fileName,
  loading,
  validating,
  error,
  success,
  warning,
  sourceData
}
```

衍生能力：
- `hasFile`
- `hasValidSources`
- `stats`

`sourceData` 目前可能包含：
- `total`
- `dedupCount`
- `duplicates`
- `duplicateUrls`
- `formatInvalid`
- `deepInvalid`
- `validCount`
- `dedupedSources`
- `failedGroups`
- `failedCategories`
- `fileStats`
- `urlStats`
- `ruleTypeStats`

## 3. 主页面职责

[frontend/src/App.vue](../frontend/src/App.vue)

负责：
- 切换 `single / batch / search`
- 触发普通接口和 SSE 接口
- 维护 `EventSource`
- 响应完成/取消/异常
- 组织统计、失败详情、下载导出

## 4. 组件清单

### 4.1 输入与配置组件

- `FileUpload.vue`
  - 文件选择
  - 拖拽上传
  - 向父组件发出 `upload`

- `UrlInput.vue`
  - 输入在线 URL
  - 一键粘贴
  - 发出 `fetch`、`paste`

- `ConcurrencySelector.vue`
  - 并发设置
  - `v-model`

- `TimeoutSelector.vue`
  - 超时设置
  - `v-model`

- `FilterOptions.vue`
  - 过滤正版/听书/漫画/影视
  - `v-model`

### 4.2 触发型组件

- `ActionButtons.vue`
  - `dedup`
  - `full-validate`

- `BatchProcessor.vue`
  - 管理批量文件/URL
  - 发出 `batch-process`
  - 支持父组件回写 `fileStats` / `urlStats`

- `SearchValidator.vue`
  - 提供 `search / explore / both`
  - 搜索关键词输入
  - 并发和超时配置
  - 发出 `start-validation`

### 4.3 结果与反馈组件

- `ValidationProgress.vue`
  - 展示 `processed / total / valid / invalid`
  - 展示当前 URL、当前名称、耗时、预计剩余时间
  - 发出 `cancel`

- `StatsCard.vue`
  - 单张卡片

- `StatsCardGroup.vue`
  - 汇总显示：
    - 总数
    - 去重后
    - 重复数
    - 格式失效
    - 深度失效

- `DuplicateUrls.vue`
  - 展示重复 URL 及出现次数

- `FailedSources.vue`
  - 以分类形式展示失败数据
  - 当前分类来源于后端 `failedCategories`

- `DownloadButton.vue`
  - 导出有效书源

- `MessageAlert.vue`
  - 错误/成功/警告提示

- `ThemeToggle.vue`
  - 亮色/暗色切换

## 5. API 交互层

[frontend/src/api/sources.js](../frontend/src/api/sources.js)

当前封装的接口分为：
- 基础解析
- SSE 深度校验
- 批量处理
- 搜索校验

EventSource 构造：
- `getProgressEventSource(sessionId)`
- `getSearchProgressEventSource(sessionId)`

## 6. 三种模式的数据流

### 6.1 单个处理

```text
FileUpload / UrlInput
  → parseFile / parseUrl
  → sourceData
  → StatsCardGroup / DuplicateUrls / DownloadButton
```

### 6.2 全部校验

```text
点击“全部校验”
  → startValidation / startValidationFromData
  → EventSource 监听进度
  → completed 后组装 resultData
  → sourceData 刷新页面
```

### 6.3 批量处理

```text
BatchProcessor
  → parseBatchFiles / parseBatchUrls
  或
  → startBatchFilesValidation / startBatchUrlsValidation
  → 返回整体结果 + 来源级统计
```

### 6.4 搜索校验

```text
SearchValidator
  → startSearchValidation
  → getSearchProgressEventSource
  → completed 后返回 validSources / failedGroups / failedCategories / ruleTypeStats
```

## 7. 组件设计原则

- 新组件使用 `<script setup>`
- 状态尽量收口在 `App.vue + useSources.js`
- 业务流程通过 API 层统一访问后端
- 复杂展示交给独立组件，避免 `App.vue` 模板继续膨胀

## 8. 当前前端风险

- `App.vue` 已经承担了大量流程控制和结果组装逻辑，后续适合继续拆分。
- SSE 完成态的结果组装逻辑在单个处理、批量处理、搜索校验中存在重复。
