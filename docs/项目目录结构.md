# 项目目录结构

本文档描述当前仓库的真实结构，以代码实现为准。

## 1. 根目录

```text
shuyuanquchong/
├── backend/                       # FastAPI 后端
├── frontend/                      # Vue 3 前端
├── docs/                          # 项目文档与会话记录
├── Dockerfile                     # Docker 镜像构建
├── docker-compose.yml             # Docker 编排
├── start.bat                      # Windows 一键启动
├── stop.bat                       # Windows 停止脚本
├── start-server.sh                # Linux 启动脚本
├── stop-server.sh                 # Linux 停止脚本
├── deploy.sh                      # Linux 部署脚本
├── deploy.bat                     # Windows 部署脚本
├── README.md
└── CLAUDE.md
```

## 2. 后端目录

```text
backend/
├── app/
│   ├── __init__.py
│   ├── main.py                    # FastAPI 应用入口，生产环境静态资源托管
│   ├── config.py                  # 全局配置
│   ├── api/
│   │   ├── __init__.py            # 路由聚合
│   │   ├── health.py              # 健康检查
│   │   └── sources.py             # 主业务路由
│   ├── models/
│   │   ├── __init__.py
│   │   ├── request.py             # 请求模型
│   │   └── response.py            # 响应模型
│   └── services/
│       ├── __init__.py
│       ├── parser.py              # 书源数组提取
│       ├── deduper.py             # URL 标准化与去重
│       ├── validator.py           # 格式校验、深度校验、分组回写
│       ├── filter.py              # 类型/关键词/域名过滤
│       ├── search_validator.py    # 搜索/发现规则校验
│       ├── session_manager.py     # SSE 会话管理
│       ├── url_security.py        # 远程 URL 安全校验
│       └── js_processor.py        # JS 规则解析辅助
├── tests/
│   ├── __init__.py
│   ├── conftest.py
│   ├── test_parser.py
│   ├── test_deduper.py
│   ├── test_validator.py
│   ├── test_filter.py
│   ├── test_api.py
│   ├── test_search_validator.py
│   └── test_url_security.py
├── requirements.txt
└── requirements-dev.txt
```

### 2.1 后端关键入口

- `backend/app/main.py`
  - 创建 FastAPI 应用
  - 配置 CORS
  - 注册 `/api`
  - 在 `frontend/dist` 存在时托管前端静态文件

- `backend/app/api/sources.py`
  - 单个文件/URL 解析
  - SSE 深度校验
  - 批量文件/批量 URL 处理
  - 搜索校验
  - 取消校验

### 3.1 前端主结构

- `App.vue`
  - 三种模式：单个处理、批量处理、搜索校验
  - 控制 SSE 连接、进度展示、导出和状态切换

- `api/sources.js`
  - 单个处理接口
  - 批量处理接口
  - SSE EventSource 构造
  - 搜索校验接口

- `composables/useSources.js`
  - 管理 `file`、`loading`、`validating`、`sourceData`
  - 为统计卡片生成衍生数据

## 5. 推荐阅读顺序

如果要快速接手开发，建议按这个顺序读：

1. `README.md`
2. `CLAUDE.md`
3. `backend/app/main.py`
4. `backend/app/api/sources.py`
5. `backend/app/services/*.py`
6. `frontend/src/App.vue`
7. `frontend/src/api/sources.js`
8. `docs/sessions/` 中最近几份会话记录
