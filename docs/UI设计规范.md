# UI 设计规范文档

## 1. 设计概述

### 1.1 设计理念
- **简洁高效**：用户操作流程最小化，核心功能一目了然
- **信息清晰**：数据统计直观展示，状态反馈及时明确
- **现代美观**：采用扁平化设计，配色柔和舒适

### 1.2 目标用户画像
| 用户类型 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 普通用户 | 技术能力一般，追求便捷 | 快速处理书源文件 |
| 书源维护者 | 有一定技术基础，频繁使用 | 批量处理、在线获取 |
| 技术爱好者 | 关注细节，追求效率 | 可靠性、性能 |

### 6.2 响应式断点
| 设备类型 | 断点 | 布局调整 |
|---------|------|---------|
| 大屏桌面 | ≥1200px | 两个操作区域并排 |
| 小屏桌面 | 768px-1199px | 两个操作区域并排，间距缩小 |
| 平板 | 576px-767px | 操作区域堆叠 |
| 手机 | <576px | 全宽布局，字号缩小 |

## 7. 交互动效

### 7.1 过渡动画
```css
/* 基础过渡 */
transition: all 0.3s ease;

/* 数字增长动画 */
@keyframes countUp {
  from { opacity: 0; transform: translateY(10px); }
  to { opacity: 1; transform: translateY(0); }
}

/* 卡片进入动画 */
@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```

### 7.2 加载状态
```
加载中状态:
┌─────────────────────────────────────────┐
│                                         │
│           ◠ ◡ ◠ (旋转动画)              │
│           正在解析中...                  │
│                                         │
└─────────────────────────────────────────┘
```

### 7.3 按钮状态
| 状态 | 视觉表现 |
|------|---------|
| 默认 | 正常颜色 |
| 悬停 | 颜色加深 10% |
| 点击 | 颜色加深 20% |
| 禁用 | 透明度 50%，光标禁用 |
| 加载 | 显示旋转图标，禁用点击 |

## 8. 图标规范

### 8.1 使用的图标
| 图标名称 | 用途 | 建议来源 |
|---------|------|---------|
| 📚 | Logo/标题 | Emoji |
| 📁 | 上传区域 | Emoji |
| ⬇️ | 下载按钮 | Emoji |
| ✅ | 成功状态 | Emoji |
| ❌ | 错误状态 | Emoji |
| ⚠️ | 警告状态 | Emoji |
| 🔗 | 链接图标 | Emoji |

### 8.2 图标尺寸
- 标题图标：24px
- 按钮图标：16px
- 提示图标：16px

---

## 9. 无障碍设计

### 9.1 对比度
- 正文文字与背景对比度 ≥ 4.5:1
- 大标题与背景对比度 ≥ 3:1

### 9.2 焦点状态
- 所有交互元素支持键盘导航
- 焦点元素有明显的外轮廓线（outline）

### 9.3 语义化标签
- 使用正确的 HTML5 语义标签
- 为图片提供 alt 属性
- 为按钮提供 aria-label

---

## 10. 设计稿导出规范

### 10.1 资源命名
```
图标: icon_[name]@[size].png
按钮: btn_[type]_[state].png
卡片: card_[name].png
```

### 10.2 切图要求
- 格式：PNG-24（透明背景）或 SVG
- 倍率：@1x, @2x, @3x（适配高清屏）
