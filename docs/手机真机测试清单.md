# 手机真机测试清单

本文档用于验证“导出到阅读 App”与移动端适配相关功能，适合在本地局域网环境下执行。

---

## 1. 测试目标

本轮真机测试重点验证：

1. 手机浏览器访问页面是否正常
2. 手机端布局与交互是否可用
3. “导出到阅读 App”是否能正常唤起阅读
4. 阅读 App 是否能正确导入书源
5. 临时导出链接是否稳定可用

---

## 2. 测试前准备

### 2.1 环境要求

- 电脑与手机连接同一 Wi-Fi
- 手机已安装阅读 App
- 电脑本地已拉起前后端服务
- 电脑防火墙允许测试端口访问

### 2.2 启动方式

后端：

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

前端：

```bash
cd frontend
npm install
npm run dev -- --host 0.0.0.0
```

手机访问地址：

```text
http://电脑局域网IP:5173
```

说明：

- 不要使用 `localhost`
- 如果手机无法访问，优先检查：
  - 电脑是否连接同一局域网
  - 防火墙是否拦截 5173 / 8000
  - Vite 是否使用了 `--host 0.0.0.0`

- [ ] 临时 JSON 链接可见且能点击

### 4.3 进度区

- [ ] 进度条显示正常
- [ ] 有效数 / 失败数 / 耗时 / 预估剩余时间不重叠

- [ ] 批量文件列表能查看长文件名

### 5.1 结果生成

- [ ] 成功得到有效书源结果

- [ ] 点击“导出到阅读 App”后浏览器尝试唤起阅读

- [ ] 阅读 App 中实际导入的书源数量正确

## 8. 常见排查方向

### 8.1 手机打不开页面

- 检查电脑与手机是否在同一 Wi-Fi
- 检查访问地址是否误用了 `localhost`
- 检查 5173 / 8000 端口是否被防火墙拦截

### 8.2 阅读 App 未被唤起

- 检查手机是否已安装阅读 App
- 检查浏览器是否拦截 `legado://` Scheme
- 尝试直接点击临时 JSON 链接确认导出链路是否正常

### 8.3 阅读导入失败

- 检查临时 JSON 链接在手机浏览器中是否可正常打开
- 检查链接是否过期
- 检查后端服务是否仍在运行

### 8.4 页面布局异常

- 记录浏览器名称和版本
- 截图问题区域
- 优先说明是“溢出”“换行异常”“按钮难点按”还是“内容遮挡”
