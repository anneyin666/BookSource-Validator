# 网站功能规划文档

## 1. 项目概述

### 1.1 项目名称

**阅读书源·净源工坊** - 开源阅读 App 书源去重校验工具

### 1.2 项目定位

为开源阅读 App 用户提供便捷的书源文件处理服务，解决书源文件中存在的重复、失效问题，提升阅读体验。

### 1.3 目标用户

- 开源阅读 App 用户
- 书源分享者
- 需要批量管理书源的用户

## 2. 核心功能模块

### 2.1 数据输入模块

#### 2.1.1 文件上传

| 功能项 | 描述 |
|--------|------|
| 支持格式 | JSON 格式书源文件 |
| 文件大小 | 最大 10MB |
| 上传方式 | 拖拽上传 / 点击选择 |
| 进度显示 | 显示上传进度 |

#### 2.1.2 URL 解析

| 功能项 | 描述 |
|--------|------|
| 协议支持 | HTTP / HTTPS |
| 超时时间 | 30 秒 |
| 数据格式 | 自动识别 JSON 格式 |
| 错误处理 | 超时、格式错误提示 |

### 2.2 处理模式模块

#### 2.2.1 只查重复模式

**功能描述**：快速去重，基于 `bookSourceUrl` 字段进行 URL 标准化去重

**处理流程**：
```
输入数据 → 提取书源 → URL标准化去重 → 格式校验 → 设置分组 → 输出结果
```

**URL标准化规则**：
- 转换为小写
- 去除尾部斜杠
- 去除首尾空白字符

**适用场景**：快速处理大量书源，仅需去重

#### 2.2.2 全部校验模式

**功能描述**：去重 + 格式校验 + 深度校验

**处理流程**：
```
输入数据 → 提取书源 → URL标准化去重 → 格式校验 → 深度校验 → 设置分组 → 输出结果
```

**深度校验规则**：
- 使用手机端 User-Agent 请求
- HTTP HEAD 方法检测 URL 可访问性
- 超时时间 10 秒
- 最大重定向 5 次

**适用场景**：确保书源可用性，筛选有效书源

### 2.3 结果展示模块

#### 2.3.1 统计卡片

| 统计项 | 描述 |
|--------|------|
| 总书源数 | 输入的书源总数 |
| 去重后 | 去重后的数量 |
| 重复数 | 被去除的重复数量 |
| 格式失效 | URL 格式无效的数量 |
| 深度失效 | 深度校验失效的数量（仅全部校验模式） |

#### 2.3.2 重复URL展示

- 可折叠的重复 URL 列表
- 显示每个 URL 出现次数
- 最多展示前 20 个重复 URL

#### 2.3.3 消息提示

| 类型 | 场景 |
|------|------|
| 成功 | 处理成功，显示有效书源数量 |
| 警告 | 发现失效书源 |
| 错误 | 文件格式错误、网络错误等 |

### 2.4 数据导出模块

#### 2.4.1 JSON 下载

| 功能项 | 描述 |
|--------|------|
| 文件格式 | JSON |
| 编码 | UTF-8 |
| 格式化 | 2 空格缩进 |
| 文件名 | `阅读书源_去重有效_YYYY-MM-DD.json` |

#### 2.4.2 分组设置

校验后的书源自动设置分组字段：
```
格式：YYYY-MM-DD校验XXX条书源
示例：2026-03-11校验98条书源
```

## 4. 性能指标

### 4.1 响应时间

| 操作 | 目标时间 |
|------|----------|
| 文件上传解析 | < 3s (1000条) |
| URL获取解析 | < 5s (含网络请求) |
| 只查重复 | < 5s (1000条) |
| 全部校验 | 根据书源数量和网络状况 |

### 4.2 并发能力

- 支持多用户同时使用
- 深度校验采用异步并发处理

### 4.3 数据限制

| 限制项 | 值 |
|--------|------|
| 最大文件大小 | 10MB |
| 请求超时 | 30s |
| 深度校验超时 | 10s/URL |
| 最大重定向 | 5 次 |

## 5. 安全考虑

### 5.1 输入验证

- 文件类型验证（仅 JSON）
- 文件大小限制
- URL 协议限制（HTTP/HTTPS）

### 5.2 请求安全

- CORS 跨域配置
- 请求超时保护
- 重定向次数限制

### 5.3 数据安全

- 不存储用户上传文件
- 不记录书源内容
- 处理完成立即释放内存
