# 系统架构设计

## 1. 系统定位

本项目是一个前后端分离的 Web 工具，面向开源阅读 App 书源整理场景，当前能力覆盖：
- 书源导入
- URL 去重
- 格式校验
- 深度校验
- 批量处理
- 搜索/发现规则校验
- SSE 实时进度
- 导出有效/失败结果

## 2. 技术栈

### 2.1 前端

| 层级 | 技术 |
|------|------|
| 框架 | Vue 3 |
| 语言 | JavaScript |
| UI | Element Plus |
| 请求 | Axios |
| 构建 | Vite |

### 2.2 后端

| 层级 | 技术 |
|------|------|
| 框架 | FastAPI |
| 语言 | Python 3.10+ |
| 模型 | Pydantic |
| HTTP | httpx |
| 服务 | Uvicorn |

## 3. 总体架构

```text
浏览器
  └─ Vue 3 单页应用
       ├─ 单个处理
       ├─ 批量处理
       ├─ 搜索校验
       └─ EventSource 监听 SSE
            │
            ▼
       FastAPI
       ├─ health 路由
       ├─ sources 主业务路由
       ├─ service 层
       │   ├─ ParserService
       │   ├─ DeduperService
       │   ├─ ValidatorService
       │   ├─ FilterService
       │   ├─ SearchValidatorService
       │   ├─ UrlSecurityService
       │   └─ SessionManager
       └─ httpx 对外请求
```

## 4. 后端分层

### 4.1 入口层

[backend/app/main.py](../backend/app/main.py)

职责：
- 创建 FastAPI 应用
- 配置 CORS
- 注册 `/api`
- 在生产环境托管 `frontend/dist`

### 4.2 路由层

[backend/app/api/sources.py](../backend/app/api/sources.py)

职责：
- 参数接收与基础校验
- 文件/URL 数据读取
- 调用服务层
- 组装普通 JSON 响应或 SSE 响应

当前接口分为四组：
- 基础解析：文件、URL
- 深度校验：开始、从已有数据开始、SSE 进度、取消
- 批量处理：批量文件、批量 URL
- 搜索校验：开始、SSE 进度

### 4.3 服务层

#### ParserService

[backend/app/services/parser.py](../backend/app/services/parser.py)

职责：
- 从数组、`sources`、`data`、`list`、嵌套对象中提取书源数组

#### DeduperService

[backend/app/services/deduper.py](../backend/app/services/deduper.py)

职责：
- 标准化 `bookSourceUrl`
- 进行去重
- 返回重复 URL 统计

URL 标准化目前包括：
- 小写化
- 去首尾空白
- 去尾部斜杠
- 去 `#` 后作者信息
- 去直接拼接在 URL 后的中文作者名

#### ValidatorService

[backend/app/services/validator.py](../backend/app/services/validator.py)

职责：
- 格式校验
- 深度校验
- 分组回写

深度校验策略：
- 默认手机端请求头
- HEAD 优先，失败回退 GET
- 200-399 有效
- 403 视为有效
- 支持重试
- 超时选项：15/30/45/60

#### FilterService

[backend/app/services/filter.py](../backend/app/services/filter.py)

职责：
- 按 `bookSourceType`
- 按名称/备注关键词
- 按正版站域名

#### SearchValidatorService

[backend/app/services/search_validator.py](../backend/app/services/search_validator.py)

职责：
- 构造搜索/发现请求
- 处理相对路径、多行 JSON 配置、部分 JS 规则
- 评估搜索/发现结果是否有效
- 对失败原因进行分类

#### UrlSecurityService

[backend/app/services/url_security.py](../backend/app/services/url_security.py)

职责：
- 校验远程 URL 是否允许被服务端访问
- 拦截 `localhost`、`.local`
- 拦截回环、私网、链路本地和保留地址

#### SessionManager

[backend/app/services/session_manager.py](../backend/app/services/session_manager.py)

职责：
- 为 SSE 任务创建会话
- 保存进度状态
- 保存完成结果
- 支持取消任务

## 5. 前端结构

### 5.1 页面入口

[frontend/src/App.vue](../frontend/src/App.vue)

页面包含三种模式：
- `single`：单个处理
- `batch`：批量处理
- `search`：搜索校验

### 5.2 数据与控制流

```text
用户操作
  └─ App.vue / 子组件事件
       └─ api/sources.js
            └─ 后端接口
                 └─ service 层处理
                      └─ 普通 JSON 或 SSE
                           └─ useSources.js 更新状态
                                └─ 统计、失败列表、下载按钮刷新
```

### 5.3 关键前端组件

- `FileUpload.vue`：上传和拖拽
- `UrlInput.vue`：在线链接解析和一键粘贴
- `ActionButtons.vue`：只查重复 / 全部校验
- `BatchProcessor.vue`：批量文件 / URL 输入和触发
- `SearchValidator.vue`：搜索、发现、搜索+发现配置
- `ValidationProgress.vue`：SSE 进度展示与取消
- `FailedSources.vue`：失败分类展示
- `StatsCardGroup.vue`：统计展示
- `ThemeToggle.vue`：主题切换

## 6. 核心处理流程

### 6.1 单个处理

```text
上传文件/输入 URL
  → 解析 JSON
  → 提取书源数组
  → 去重
  → 类型过滤
  → 格式校验
  → 可选深度校验
  → 写入 bookSourceGroup
  → 返回统计、失败信息、有效书源
```

### 6.2 SSE 深度校验

```text
前端请求 start
  → 后端完成去重/过滤/格式校验
  → SessionManager 创建会话
  → 前端连接 /progress/{session_id}
  → 后端并发校验并持续推送进度
  → 完成后推送 validSources / failedGroups
```

### 6.3 批量处理

```text
多文件或多 URL
  → 逐个解析来源
  → 合并 all_sources
  → 统一去重/过滤/校验
  → 返回整体结果 + fileStats/urlStats
```

### 6.4 搜索校验

```text
上传书源文件
  → 提取书源
  → 创建搜索校验会话
  → 按 search/explore/both 校验
  → 输出 validSources、failedGroups、failedCategories、ruleTypeStats
```

## 7. 生产部署方式

当前支持两种方式：
- 开发环境：Vite `5173` + Uvicorn `8000`
- 生产环境：前端构建后由 FastAPI 直接托管静态资源，或由外层 Nginx 反代

## 8. 当前架构特征

- 后端无数据库，主要是内存态处理
- 会话状态保存在进程内存
- SSE 适合当前校验进度场景
- `sources.py` 目前承载了较多流程编排逻辑，是后续最值得拆分的模块之一
- 在线 URL 获取已实现基础安全校验
