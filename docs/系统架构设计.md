# 系统架构设计文档

## 1. 架构概述

### 1.1 系统定位
开源阅读书源去重校验工具是一个轻量级的 Web 应用，采用前后端分离架构，提供书源文件的解析、去重、校验和导出功能。

### 1.2 设计原则
- **简单优先**：功能单一明确，避免过度设计
- **前后端分离**：前端负责 UI 交互，后端负责数据处理
- **无状态服务**：后端不依赖 session，所有请求独立处理
- **渐进增强**：核心功能无需登录，后续可扩展用户系统

### 4.2 模块职责

#### 4.2.1 API 层 (app/api/)
- **health.py**: 健康检查端点
- **sources.py**: 书源解析相关端点
  - 接收请求参数
  - 调用服务层处理
  - 返回格式化响应

#### 4.2.2 服务层 (app/services/)
- **parser.py**: JSON 解析服务
  - 解析文件内容
  - 兼容多种书源格式
  - 异常处理

- **deduper.py**: 去重服务
  - 基于 bookSourceUrl 去重
  - 保留首次出现的书源
  - 返回去重结果

- **validator.py**: 校验服务
  - **格式校验**：校验 bookSourceUrl 存在性和 URL 格式合法性
  - **深度校验**：模拟手机端请求访问书源URL，检测实际可用性
  - 使用手机端 User-Agent 头信息
  - 单个书源校验超时 10 秒
  - HTTP 状态码 200-399 视为有效

- **group_service.py**: 分组服务
  - 设置书源分组字段
  - 格式：`YYYY-MM-DD校验XXX条书源`

#### 4.2.3 模型层 (app/models/)
- 定义 Pydantic 模型
- 请求/响应数据验证
- 自动生成 OpenAPI 文档

- 去重算法优化（使用 set）

```python
# 高效去重算法
def dedupe_sources(sources: list[dict]) -> list[dict]:
    seen = set()
    result = []
    for source in sources:
        url = source.get('bookSourceUrl', '').strip()
        if url and url not in seen:
            seen.add(url)
            result.append(source)
    return result
```
