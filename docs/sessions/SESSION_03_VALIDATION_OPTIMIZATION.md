# SESSION_03 校验优化与失败分析

## 会话目标

优化深度校验功能，添加重试机制、超时配置、失败原因分组等高级功能。

## 完成任务

### 1. 校验优化

| 功能 | 说明 |
|------|------|
| 重试机制 | 网络波动时自动重试 1-2 次 |
| 403 处理 | 忽略 Cloudflare 防护导致的 403 错误 |
| 超时配置 | 支持 15/30/45/60 秒可选，默认 30 秒 |

### 2. 失败书源分析

- 按失败原因分组显示（超时、连接失败、HTTP错误等）
- 支持展开/折叠查看详细书源列表
- 支持导出失败书源 JSON 文件

### 3. 历史记录

- 使用 localStorage 保存用户设置
- 记住并发数和超时时间选择
- 下次打开自动恢复上次设置

## 技术决策

### 后端改进

1. **重试机制**
   - 最大重试次数：2 次
   - 重试间隔：1 秒
   - 重试条件：超时、连接失败等网络错误

2. **失败原因分类**
   - 超时
   - 连接失败
   - 连接超时
   - HTTP 状态码错误
   - 读取/写入错误

3. **403 处理**
   - 403 状态码视为有效（Cloudflare 防护）

### 前端改进

1. **新增 TimeoutSelector 组件**
   - 提供 15/30/45/60 秒选项
   - 双向绑定 v-model

2. **失败分组展示**
   - 可展开/折叠的分组列表
   - 显示失败原因和数量
   - 支持导出失败书源

3. **设置持久化**
   - localStorage 存储
   - 自动恢复用户偏好

## 使用说明

### 超时时间选择

在文件上传后，选择合适的超时时间：
- **15 秒**：快速校验，适合网络良好的情况
- **30 秒**：默认值，平衡速度和准确性
- **45 秒**：网络较慢时推荐
- **60 秒**：最大超时，确保慢速网站也能检测

### 失败书源导出

校验完成后，在失败分析区域：
1. 点击分组展开查看详细书源
2. 点击"导出失败书源"按钮下载 JSON 文件

### 历史记录

系统自动记住您的设置：
- 并发数选择
- 超时时间选择

下次打开页面时自动恢复。
